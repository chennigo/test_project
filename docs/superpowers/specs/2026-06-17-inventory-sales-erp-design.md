# 货物管理与销售一体化系统 — 设计规格

**日期：** 2026-06-17  
**状态：** 已批准  
**方案：** 单体分层 + 分阶段交付（方案一）

---

## 1. 项目概述

### 1.1 目标

构建一套面向公司内部使用的 ERP 系统，集成货物（库存）管理与销售功能，支持多仓库运营，员工通过内网浏览器访问。

### 1.2 约束与偏好


| 维度    | 决策                                                      |
| ----- | ------------------------------------------------------- |
| 系统类型  | 内部 ERP，浏览器访问                                            |
| 规模    | 多仓库，6～20 并发用户                                           |
| 部署    | 公司内网一台服务器（如 `http://192.168.x.x:8080`）                  |
| 技术栈   | Spring Boot 3 + Java 17 + Vue 3 + Element Plus + SQLite |
| 第一版模块 | 商品、库存、调拨、销售、报表、权限（**不含采购**）                             |
| 库存粒度  | 批次管理（批次号、生产日期）；后续可扩展条码/保质期                              |
| 销售    | 客户档案 + 历史订单（不含信用/账期）                                    |
| 前端设计  | 实现 UI 时调用 `frontend-design` 技能，避免模板化后台风格                |


### 1.3 角色与权限


| 角色            | 说明                                    |
| ------------- | ------------------------------------- |
| **超级管理员**     | 全部权限，含用户/角色/仓库配置                      |
| **仓管/销售（合一）** | 负责本仓入库、出库、盘点、调拨、开销售单、查库存；**仅限被分配的仓库** |
| **财务/统计**     | 只读报表，不能修改业务数据                         |


数据隔离：每个员工只能看到和操作被分配的仓库。

## 2. 整体架构

```
systelm/
├── backend/          # Spring Boot 后端
│   ├── controller/   # REST API
│   ├── service/      # 业务逻辑
│   ├── repository/   # 数据访问
│   ├── entity/       # 实体
│   └── security/     # JWT 认证与权限
├── frontend/         # Vue 3 前端
│   ├── views/
│   ├── components/
│   └── api/
└── data/
    └── systelm.db    # SQLite 数据库文件
```

- 前后端分离开发；上线时前端打包为静态资源，与后端 JAR 一并部署。
- 所有库存变动经单据驱动，不直接修改库存快照表。
- SQLite 适用于 6～20 人内网场景；未来可迁移 MySQL/PostgreSQL。

### 分阶段交付


| 阶段      | 范围                     | 验收标准                |
| ------- | ---------------------- | ------------------- |
| **阶段1** | 登录权限、仓库、商品/批次、入库/出库/盘点 | 可登录、建仓、录商品、完成入出库与盘点 |
| **阶段2** | 调拨、客户档案、销售开单           | 可调拨、开销售单、查客户历史      |
| **阶段3** | 报表、库存预警                | 可看销售/库存/利润报表及低库存提醒  |

## 3. 数据模型

### 3.1 核心实体


| 表                              | 用途     | 关键字段                                                            |
| ------------------------------ | ------ | --------------------------------------------------------------- |
| user                           | 员工     | username, password_hash, role_id, status                        |
| role                           | 角色     | name (admin / warehouse_sales / finance)                        |
| user_warehouse                 | 用户-仓库  | user_id, warehouse_id                                           |
| warehouse                      | 仓库     | name, address, status                                           |
| product_category               | 分类     | name, parent_id                                                 |
| product                        | 商品     | name, sku, category_id, cost_price, sale_price, unit, min_stock |
| product_batch                  | 批次     | product_id, batch_no, production_date                           |
| stock                          | 库存快照   | warehouse_id, product_id, batch_id, quantity                    |
| stock_in / stock_in_item       | 入库单及明细 | warehouse_id, items(product, batch, qty)                        |
| stock_out / stock_out_item     | 出库单及明细 | warehouse_id, type, items                                       |
| stock_check / stock_check_item | 盘点单及明细 | warehouse_id, book_qty, actual_qty, diff                        |
| transfer / transfer_item       | 调拨单及明细 | from_warehouse, to_warehouse, status, items                     |
| customer                       | 客户     | name, contact, phone, address                                   |
| sales_order / sales_order_item | 销售单及明细 | customer_id, warehouse_id, total, status, items(batch)          |


### 3.2 库存规则

1. **入库**：选仓库 → 商品 + 批次（可新建）→ 增加 `stock.quantity`。
2. **出库**：选仓库 → 商品 + 批次 → 校验库存 ≥ 出库量 → 扣减。
3. **销售**：确认销售单 → 自动生成出库记录并扣减对应批次库存。
4. **调拨**：调出仓扣减 → 调入仓增加（同批次号；调入仓无该批次则新建 stock 行）。
5. **盘点**：录入实盘数 → 与账面差值生成盘盈/盘亏调整。

## 4. 功能模块

### 4.1 阶段1

- 登录（JWT）
- 用户管理（超管）：用户 CRUD、角色分配、仓库分配
- 仓库管理（超管）
- 商品管理：CRUD、分类、进价/售价、最低库存
- 批次：随商品或入库时维护
- 入库 / 出库 / 盘点

### 4.2 阶段2

- 仓库调拨
- 客户管理（CRUD + 历史订单）
- 销售开单（选客户、发货仓、商品+批次+数量+单价）
- 库存查询（按仓/商品/批次）

### 4.3 阶段3

- 库存报表（数量、金额）
- 销售报表（日/月/商品/客户）
- 库存预警（低于 `min_stock`）
- 利润分析（销售额 − 进价成本）

### 4.4 界面布局

- 顶栏：Logo、系统名、当前用户、退出
- 侧栏：按角色动态菜单
- 主区：列表 + 搜索 + 操作（新增/编辑/详情）

**UI 实现要求：** 进入前端页面设计与开发阶段时，必须调用 `frontend-design` 技能，基于「内部多仓 ERP、仓管与销售一线使用」的场景制定视觉方案（配色、字体、布局），避免通用 AI 后台模板感；业务组件仍可使用 Element Plus，但需在 `frontend-design` 指导下做主题与布局定制。

## 5. 权限矩阵


| 功能         | 超管   | 仓管/销售  | 财务   |
| ---------- | ---- | ------ | ---- |
| 用户/角色/仓库配置 | ✅    | ❌      | ❌    |
| 商品管理       | ✅ 读写 | 只读     | 只读   |
| 入库/出库/盘点   | ✅    | ✅ 仅分配仓 | ❌    |
| 调拨         | ✅    | ✅ 仅分配仓 | ❌    |
| 销售/客户      | ✅    | ✅      | 客户只读 |
| 库存查询       | ✅ 全部 | ✅ 仅分配仓 | 只读   |
| 报表/预警      | ✅    | 预警仅分配仓 | ✅ 只读 |


**实现：** 后端每个含 `warehouse_id` 的接口校验当前用户是否拥有该仓库；前端下拉与列表按 `user_warehouse` 过滤。

## 6. 部署

1. 打包：`backend` → JAR；`frontend` → `npm run build`
2. 服务器安装 Java 17，放置 JAR、静态资源、`data/systelm.db`
3. 启动：`java -jar systelm.jar`（默认 8080）
4. 员工访问：`http://<内网IP>:8080`
5. 备份：定期复制 `systelm.db`

---

## 7. 非目标（第一版不做）

- 采购管理（采购单、供应商）
- 客户信用/账期/欠款
- 条码扫码（数据结构预留，UI 后续迭代）
- 保质期与临期预警（批次表预留 `production_date`，规则后续加）
- 公网访问 / 多租户

## 8. 测试策略

- 后端：JUnit 5 + 关键业务单测（入库扣减、调拨、销售出库、仓库权限拦截）
- 前端：核心流程手工验收清单（按阶段）
- 遵循 Superpowers TDD：实现计划阶段为每个任务编写红/绿测试

## 9. 后续扩展路径

- 条码：商品 `barcode` 字段 + 扫码入库/出库页
- 保质期：`expiry_date` + 临期报表
- 采购模块：供应商 + 采购单 → 入库联动
- 数据库：SQLite → MySQL（换数据源，业务层不变）
